# ASCII Diagram Conventions (standard entities)

All cluster diagrams in the day docs use one shared visual vocabulary so the
same entities look identical everywhere. The diagrams are generated by the
scripts in this folder; edit those, not the pasted ASCII, then regenerate.

## Tooling
- `k8sdiagram.py` — the renderer library (`cluster_map`, `sequence`).
- `gen.py` — defines the per-day focus annotations and prints each block.

```bash
python3 gen.py            # print every standardized block, labelled
python3 gen.py day16      # print one block
python3 gen.py day21-csr  # print one sequence
```

## Standard entities (always drawn the same way)

```
   CONTROL PLANE components (small tokens, left to right):
     +------------+   +------+   +-----------+   +----------------+
     | api-server |   | etcd |   | scheduler |   | controller-mgr |
     +------------+   +------+   +-----------+   +----------------+

   WORKER NODE   (kubelet | kube-proxy | runtime)
     namespace: <ns>
       POD
         CONTAINER  app
```

- **api-server** — front door; all components talk through it.
- **etcd** — key-value store of all cluster state (and Secrets).
- **scheduler** — assigns pods to nodes.
- **controller-mgr** — drives actual state toward desired (Deployments, Jobs, HPA...).
- **WORKER NODE** — always shows `kubelet | kube-proxy | runtime`.
- **namespace** — logical boundary inside the cluster.
- **POD** -> **CONTAINER** — the workload.

## Focus convention
Every "Where this fits in the cluster" diagram uses the **same** topology; the
day's relevant entity gets a `<== ...` annotation. Everything else stays
identical between docs so readers anchor on one mental picture.

## Sequence (flow / attack) diagrams
Flows use lifelines with numbered, linked steps and a trailing `STOPPED BY:` /
`DEFENSE:` / `RESULT:` line.

- Generic TLS (Day 20): actors `CLIENT`, `ATTACKER`, `SERVER`, `CA`.
- Kubernetes TLS (Day 21): actors `kubectl`, `api-server`, `etcd`, `kubelet`, `CA`, `ATTACKER`.

## Adding or changing a diagram
1. Edit the `MAPS` (cluster focus) or `SEQS` (flows) dict in `gen.py`.
2. Run `python3 gen.py <key>` and paste the block into the doc, indented to
   match the surrounding fenced code block.
3. Keep the entity labels above; do not invent new names for the same thing.
